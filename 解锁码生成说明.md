# 解锁码生成说明

## 概述

当用户首次运行工具或次数用完后，需要向其提供解锁码。解锁码基于用户的机器码生成，只能用于该特定机器。

---

## 快速开始

### 前提条件

确保你有以下文件：
- `private_key.txt` - 私钥文件（首次运行自动生成）
- `key_generator.py` - 1.0 解锁码生成器
- `key_generator_v1_1.py` - 1.1 解锁码生成器

### 生成解锁码

#### 1.0 用户

**方法1：使用 key_generator.py（推荐）**

```bash
python key_generator.py <1.0机器码>
```

**示例：**
```bash
python key_generator.py 4F0C-DDCE-0E92-1852
```

输出：
```
============================================================
           重置码生成成功
============================================================

机器码: 4F0C-DDCE-0E92-1852
重置码: inEneEaZcWzZ4Ukkyt1oAw==

============================================================
请将此重置码发送给用户
用户需要在程序提示时输入此重置码
============================================================
```

**方法2：使用 license_manager.py**

```bash
python license_manager.py --generate <1.0机器码>
```

#### 1.1 用户

1.1 机器码以 `V11-` 开头，并且按天变化。
如果用户申请 1.1 解锁码，必须发送当天界面显示的完整机器码。

**推荐方法：使用 key_generator_v1_1.py**

```bash
python key_generator_v1_1.py <1.1机器码>
```

**示例：**
```bash
python key_generator_v1_1.py V11-AAAA-BBBB-CCCC-DDDD
```

输出：
```
============================================================
        1.1 解锁码生成成功
============================================================

机器码: V11-AAAA-BBBB-CCCC-DDDD
解锁码: xxxxxxxxxxxxxxxxxxxxxx==

============================================================
请将此解锁码发送给 1.1 用户
============================================================
```

**注意：** `python license_manager.py --generate <机器码>` 当前仍按 1.0 流程生成，不用于 1.1 机器码。

---

## 验证私钥

私钥文件 `private_key.txt` 是生成解锁码的唯一凭证。

**重要：**
- 不要丢失 `private_key.txt` - 丢失后无法生成解锁码
- 不要提交到 git - 已在 .gitignore 中排除
- 不要分发给用户

**检查私钥是否存在：**
```bash
cat private_key.txt
```

如果不存在，首次运行生成器时会自动创建。

## 完整流程示例

### 场景1：1.0 用户首次使用

1. **用户运行程序**，看到提示：
   ```
   本工具需要解锁后才能使用。
   请联系我获取解锁码：
     QQ: 843115404

   您的机器码: A1B2-C3D4-E5F6-G7H8
   ```

2. **用户发送机器码**给你

3. **你生成解锁码**：
   ```bash
   python key_generator.py A1B2-C3D4-E5F6-G7H8
   ```

4. **发送解锁码给用户**

5. **用户输入解锁码**，获得30次使用次数

### 场景2：1.1 用户首次使用

1. **用户运行程序**，看到提示：
   ```
   本工具需要解锁后才能使用。
   请复制当天显示的机器码发送给我，我会为您生成解锁码。

   您的机器码: V11-AAAA-BBBB-CCCC-DDDD
   ```

2. **用户把当天显示的 1.1 机器码发送给你**

3. **你生成 1.1 解锁码**：
   ```bash
   python key_generator_v1_1.py V11-AAAA-BBBB-CCCC-DDDD
   ```

4. **发送 1.1 解锁码给用户**

5. **用户输入解锁码**，获得30次使用次数

---

### 场景3：用户次数用完

无论 1.0 还是 1.1，次数用完后的处理流程都相同：
- 用户重新提供当前版本对应的机器码
- 你按对应版本重新生成解锁码
- 用户输入后会重置为 30 次

## 常见问题

### Q: 提示 "private_key.txt 不存在"

A: 首次运行生成器时会自动创建私钥。如果误删，需要重新创建（之前生成的解锁码将失效）。

### Q: 用户说解锁码无效

A: 检查：
1. 机器码是否复制完整
2. 1.0 和 1.1 是否用了对应版本的生成脚本
3. 1.1 用户发送的是否是当天界面显示的机器码
4. 是否有多余空格
5. 私钥文件是否正确（是否被修改过）

### Q: 1.1 用户昨天申请过，今天还能继续用昨天的机器码生成吗？

A: 不建议。1.1 机器码按天变化，应该让用户发送当天界面显示的机器码，再生成当天对应的 1.1 解锁码。

### Q: 如何在多台机器上测试

A: 每台机器有不同的机器码，需要分别为每台机器生成解锁码。1.1 版本除了机器不同，日期不同也会导致显示的机器码不同。

## 技术细节

### 解锁码算法

```text
1.0 重置码 = Base64(HMAC-SHA256(去掉连字符后的机器码, 私钥)[0:16])
1.1 解锁码 = Base64(HMAC-SHA256(完整 1.1 机器码, 私钥)[0:16])
```

其中 1.1 机器码格式为：

```text
V11-XXXX-XXXX-XXXX-XXXX
```

它由设备指纹与当天日期戳共同计算，因此会按天变化。

### 安全性

- 没有私钥无法伪造解锁码
- 解锁码只能用于特定机器码
- 1.1 版本额外绑定当天显示的机器码
- 使用标准 HMAC-SHA256 算法
